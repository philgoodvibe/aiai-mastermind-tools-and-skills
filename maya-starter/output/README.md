# output/ - Where Posts Live and Move

Every post MAYA creates is a folder that moves through three states. MAYA moves the folders. You just review.

```
output/
  drafts/     Compliance-passed drafts, staged in Mai Marketing Machine, awaiting your review
  approved/   You said yes; the post is scheduled in Mai Marketing Machine
  posted/     Confirmed live on the platform
```

## The post folder

Each post is one folder, named `YYYY-MM-DD-<lane>-<slug>`, for example `2026-06-15-newsjacking-world-cup-goalkeeper`. Inside:

```
2026-06-15-newsjacking-world-cup-goalkeeper/
  post.md        The draft (template below)
  image-1.png    The visual(s), generated by the image-brand skill
```

## post.md template

```markdown
# [Post title for internal reference]

- Date: YYYY-MM-DD
- Lane: newsjacking | problem-solution | entertain-meme | repurpose
- Pillar: Relate | Reveal | Guide | Invite
- Format: photo + caption | carousel + caption
- Platform: Instagram
- Status: draft | approved | posted
- Compliance: PASS (date) or pending

## Hook options considered
1. ...
2. ...
3. ... (chosen)

## Caption
[Final caption text, including agency name and contact]

## Image
- Text on image: "..."
- Visual direction: [one line on the concept and emotion]

## Research notes
[Source links, the news item or complaint quoted, why the ICP cares]

## Compliance record
[What was checked, anything rewritten, and the rule applied]
```

## How folders move

1. A lane skill plus image-brand creates the folder in `drafts/` (only after a compliance PASS).
2. posting-mmm stages it in Mai Marketing Machine as a DRAFT and tells the owner.
3. Owner approves (in Mai Marketing Machine or in chat): MAYA schedules it and moves the folder to `approved/`, updating Status.
4. Once confirmed live: MAYA moves it to `posted/`.

A draft that fails review goes back through the pipeline, not into the trash. The feedback gets written into the skills and the brain so the same note never comes back twice.
